Introduction to Flask

Flask is a lightweight WSGI web application framework in Python. It’s designed to make getting started quick and easy, with the ability to scale up to complex applications.

Client → Server → Flask → Browser
🌐 Browser
➡️
🖥️ Flask Server
⬅️
📄 HTML/JSON

Client sends request → Flask processes → returns response (HTML/JSON)

Important: Flask is a micro-framework, but it’s powerful. It doesn’t force a particular project structure.

Why Flask?

  • Minimalistic and easy to learn.
  • Built-in development server and debugger.
  • Integrated support for unit testing.
  • RESTful request dispatching.
  • Large ecosystem of extensions.

Installing Flask

pip install flask
Use a virtual environment to avoid conflicts.

First Flask Application

from flask import Flask
app = Flask(__name__)

@app.route('/')
def hello():
    return 'Hello, Flask!'

if __name__ == '__main__':
    app.run(debug=True)
Understanding Every Line
  • Flask(__name__) – creates the app instance.
  • @app.route('/') – decorator that maps URL to function.
  • hello() – view function that returns a response.
  • app.run(debug=True) – runs the server with auto-reload.

Flask Routes & Dynamic Routes

@app.route('/user/<name>')
def user(name):
    return f'Hello, {name}!'

@app.route('/post/<int:post_id>')
def post(post_id):
    return f'Post ID: {post_id}'

Dynamic Use <type:variable> to capture values from the URL.

Static Files (CSS, JS, Images)

<link rel="stylesheet" href="{{ url_for('static', filename='style.css') }}">

Place static files in the static/ folder.

Forms (GET & POST)

from flask import request

@app.route('/login', methods=['GET', 'POST'])
def login():
    if request.method == 'POST':
        username = request.form['username']
        return f'Welcome {username}'
    return render_template('login.html')
request object contains all client data.

JSON and REST APIs

from flask import jsonify

@app.route('/api/data')
def api_data():
    return jsonify({'name': 'Flask', 'version': '2.3'})

Use jsonify() to return JSON responses. Flask makes building REST APIs a breeze.

Connecting Flask with Machine Learning Models

import joblib
model = joblib.load('model.pkl')

@app.route('/predict', methods=['POST'])
def predict():
    data = request.get_json()
    prediction = model.predict([data['features']])
    return jsonify({'prediction': prediction.tolist()})

Load your model once and use it inside route handlers.

Student Result Predictor (Mini Project)

Complete Student Result Predictor Project – a Flask app that predicts student grades based on study hours.

# app.py
from flask import Flask, render_template, request, jsonify
import joblib
import numpy as np

app = Flask(__name__)
model = joblib.load('student_model.pkl')  # trained model

@app.route('/')
def index():
    return render_template('predict.html')

@app.route('/predict', methods=['POST'])
def predict():
    hours = float(request.form['hours'])
    prediction = model.predict([[hours]])
    return render_template('result.html', hours=hours, result=prediction[0])

if __name__ == '__main__':
    app.run(debug=True)

Build a simple linear regression model with sklearn, save it, and serve predictions via Flask.

Best Practices

  • Use environment variables for configuration.
  • Organize routes with Blueprints.
  • Always handle errors gracefully.
  • Keep business logic separate from routes.

Common Errors & Solutions

  • ImportError: Flask not installed → run pip install flask
  • 404 Not Found: route not defined → check URL and decorator
  • 500 Internal Server: bug in code → use debug=True to trace
